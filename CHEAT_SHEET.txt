DLMS METER XML CONVERSION - ONE-PAGE CHEAT SHEET
================================================

PROBLEM
-------
Meter data reads successfully but XML conversion is missing:
  ✗ Instantaneous data (voltage/current/power)
  ✗ Load profile data (historical readings)

ROOT CAUSES
-----------
Instantaneous:  Meter timeout | Invalid OBIS | Access denied
Load Profile:   No capture objects | Bulk read timeout | Session dropped
                Empty buffer | Timeout before completing read

SOLUTION
--------
1. CODE: Validation methods added to Reading.java
2. TOOL: DLMSToXMLConverter.java converts TXT → XML  
3. DOCS: 4 reference documents explain DLMS format

QUICK START
-----------

Step 1: Rebuild app
  ./gradlew assembleDebug && adb install -r app/build/outputs/apk/debug/app-debug.apk

Step 2: Read meter
  Open app → Select LOAD_PROFILE mode → Click Read

Step 3: Check logs
  adb pull /sdcard/NPCL_OPTICAL_LOG.TXT
  Search for: VALIDATION_OK or VALIDATION_CRITICAL

Step 4: Convert to XML
  java DLMSToXMLConverter meter_data.TXT meter_data.xml

Step 5: Verify XML
  Check for <Instantaneous> and <LoadProfile> sections
  Look for <Warning> tags indicating missing data

VALIDATION MESSAGE MEANINGS
---------------------------

✓ VALIDATION_OK: Instantaneous data FOUND
  → Good! Data collected properly for instant read

✗ VALIDATION_WARN: Instantaneous data MAY BE MISSING
  → Check meter connection, retry read with longer timeout

✓ VALIDATION_LP: CaptureObj=true, Period=true, Buffer=true
  → Good! Load profile data collected properly

✗ VALIDATION_LP: CaptureObj=false
  → Meter rejected attr=3 (using hardcoded fallback)

✗ VALIDATION_LP_RECORDS: 0 records found
  → No historical data transferred (meter may be new)

? VALIDATION_CRITICAL: LP Buffer MISSING
  → Critical error: load profile read incomplete

OBIS CODE - DATA MAPPING
------------------------

Class | OBIS         | Attribute | What It Is              | Example Value
======|==============|===========|========================|================
0001  | 0000600100FF | 2         | Meter number (ID)       | "MgE"
0003  | 0100010200FF | 2         | VOLTAGE                 | 230.5 V
0003  | 0100020200FF | 2         | CURRENT                 | 12.3 A
0004  | 0100030700FF | 2         | POWER                   | 2.8 kW
0007  | 0100620100FF | 3,2,7     | BILLING PROFILE         | Historical bills
0007  | 0100630100FF | 3,4,7,2   | LOAD PROFILE            | Historical each hour
0008  | 0000010000FF | 2         | RTC / Clock             | Date/Time

FINDING THE PROBLEM
-------------------

In TXT file, look for lines starting with these:

INSTANT mode needs:      LOAD_PROFILE mode needs:
✓ 0001 (Meter ID)        ✓ All plus:
✓ 0003 (Register)        ✓ 0007 0100630100FF 03 (Capture objects)
✓ 0004 (Extended)        ✓ 0007 0100630100FF 04 (Capture period)
✓ 0008 (Clock)           ✓ 0007 0100630100FF 07 (Entries count)
                         ✓ 0007 0100630100FF 02 (Buffer data)

Command to check:
  grep "^0100630100FF" meter_data.TXT
  → Should show 4 lines (for attr 3, 4, 7, 2)

LOAD PROFILE RECORD COUNT
------------------------

In TXT BufferData line, look for pattern: 0212090c
  Each "0212090c" = 1 record (timestamp + values)

Count them:
  grep "0212090c" meter_data.TXT | wc -l
  → How many historical records were transferred

If 0 records:
  ✗ Meter has no history (new device)
  ✗ OR load profile empty
  ✗ OR read was incomplete → increase timeout

METER-SPECIFIC NOTES
--------------------

HPL Meters:
  • Might reject attr=3 read (uses hardcoded objects)
  • Capture period usually 30 minutes
  • Check ReadLoadSurveyData() hardcoded mapping

Secure Meters:  
  • Supports secure OBISs: 01005E5B00FF
  • Capture period usually 15 minutes (900 seconds)
  • Verify amendment configuration

Genus Meters:
  • May require authentication level 1
  • Access tier restrictions apply
  • Need secure DLMS connection

QUICK FIXES
-----------

For Missing Instantaneous:
  → Check meter is responding: grep "INSTANT_DONE" log
  → Try longer timeout: bytTimOut = (byte)5
  → Verify OBIS codes for your meter make

For Missing Load Profile:
  → Check attr=3: grep "0100630100FF 03" TXT file
  → If missing, may need hardcoded capture objects
  → Increase LP timeout: lpDeadlineMs += 120000L
  → Reduce days: Select 10 days instead of 30

For Load Profile Timeout:
  → Increase timeout: bytTimOut = (byte)10
  → Reduce days requested
  → Check USB cable quality

DEBUGGING FLOW CHART
-------------------

No validation messages?
├─→ Code not rebuilt yet → Rebuild and deploy

INSTANT data missing?
├─→ Meter not responding to instant reads
│   └─ Check USB, try longer timeout

LP CaptureObj false?
├─→ Meter rejects attr=3 read
│   └─ Using hardcoded, needs verification

LP Buffer missing?
├─→ Bulk read failed or timed out
│   └─ Check for "RLS_BULK_CHECK ok=false"
│   └─ Increase timeout or reduce days

LP Records 0?
├─→ No history in meter (normal for new meter)
│   └─ Or read was incomplete
│   └─ Try again with longer timeout

XML has <Warning>?
├─→ Converter detected missing sections
│   └─ Check which sections are missing
│   └─ Return to TXT validation step

5. Read mode attempted:        INSTANT? LOAD_PROFILE?

Include in bug report:
  • NPCL_OPTICAL_LOG.TXT (last 50 lines around VALIDATION_)
  • {MeterNo}_{Time}.TXT (first 100 lines)
  • {MeterNo}_{Time}.xml (entire file if created)
  • Phone model and Android version
  • Meter make/model and age
